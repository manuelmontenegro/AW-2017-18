Una visión global de la asignatura

Las tecnologías web utilizan una arquitectura de cliente/servidor.

El servidor es un programa que responde peticiones

El cliente realiza esas peticiones, a través de un navegador.

Ambos componentes utilizan el mismo protocolo

→ Tema 1: El protocolo HTTP

El servidor puede responder con la estructura de una página web en formato HTML...

... o con información sobre su visualización (CSS) ...

→ Tema 2: HTML y CSS

... u otro tipo de recurso (imágenes, tipos de letra, etc.)

El cliente (navegador) utiliza la información recibida para renderizar la página web y mostrarla al usuario.

Evaluación

Tanto en junio como en septiembre:

  • 10% - Ejercicios sencillos.
  • 30% - Prácticas (obligatorias).
  • 60% - Examen final en laboratorio.

Para aprobar es necesario:

  • Calificación de APTO en cada práctica.
  • Calificación de 4 sobre 10 en el examen.
  • Calificación final ponderada mayor o igual que 5.

Calificación de los ejercicios

Se realizan por parejas

Se evaluarán como APTO/NO APTO

La puntuación de los ejercicios vendrá dada por:

$$10 * \frac{\text{Num. ejercicios aptos}}{\text{Num. ejercicios propuestos}}$$

Calificación de las prácticas

  • NO APTO no recuperable
    Práctica copiada o no entregada.
  • NO APTO recuperable
    Errores graves de diseño, o funcionamiento incorrecto. Puede volver a entregarse.
  • APTO con calificación numérica
    La calificación numérica se otorga en función de las extensiones entregadas.
